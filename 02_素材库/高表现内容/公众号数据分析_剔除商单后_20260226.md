## 爆款底层逻辑拆解

### 一、标题句式规律

**句式 1：「我 + 动作 + 结果/感悟」（第一人称叙事）**
- 「我被 Gemini 骂醒了」
- 「我公众号的文章全是AI写的，但没人认出来」
- 「我用 Claude Code 造了个五人智囊团」
- 「我用 skill 帮媳妇30秒改完论文格式，我悟出了…」
- 「我用Claude Code + Skill 帮老婆跑通…」

Top20 中有 **8 篇**用第一人称叙事开头，占比 40%。

**句式 2：「数字 + 极限词 + 工具/方法」（资源型）**
- 「2026 年初最强vibe coding工具组合｜白嫖版」
- 「2025 年末最强穷鬼 vibe coding 工具组合」
- 「近期热门 Claude Skill 仓库合集」
- 「中文用户必须知道的 Skill 商店」

**句式 3：「一个 + 最小化描述 + 解决大问题」（降低门槛型）**
- 「一个 Skill，解决 90% 的信息图需求」
- 「一个最简单的 skill，把 Claude Code打造成数字分身」
- 「一招让你在 Claude code 里面爽用opus 4.5」

**句式 4：「颠覆认知 + 反转」**
- 「技术在这个时代是最廉价的壁垒」
- 「我劝你：要对 Claude skill 祛魅」
- 「别再劝小白学 AI 编程了，那是杀人诛心」（Bottom区，但句式值得注意）

### 二、情绪触发机制

| 情绪类型 | 代表标题 | 触发逻辑 |
|----------|----------|----------|
| 危机感/被骂醒 | 「我被 Gemini 骂醒了」 | 「我差点错了」比「你应该这样做」更有代入感 |
| 秘密感/内幕 | 「没人认出来，只因为这招」 | 「只因为这招」制造信息不对称 |
| 亲密感/生活化 | 「帮媳妇/老婆」系列 | 把技术落到真实生活场景，降低距离感 |
| 稀缺感 | 「中文用户必须知道」「白嫖版」 | 「必须知道」暗示你不知道就亏了 |
| 好奇心 | 「刘润和罗振宇为我的项目吵了一架」 | 具体人名 + 冲突场景，强迫点开 |

### 三、关键词规律

**高频出现词（Top20 vs Bottom20 对比）：**

Top20 高频：`skill`（12次）、`Claude Code`（8次）、`我`（8次）、`最强/最简单/最高级`（6次）、`帮`（4次）

Bottom20 高频：`框架`、`方法论`、`拆解`、`闭门会`、`播客`

**结论：**
- 「工具 + 我的实操」比「方法论 + 拆解」更能打开率
- 「帮别人解决了什么」比「我学到了什么框架」更有吸引力
- 具体工具名（skill/Claude Code/vibe coding）比抽象概念（上下文/提示词工程）更好

## 可操作结论

**做什么：**
1. 「我 + 帮谁 + 解决了什么具体问题 + 悟出了什么」是最稳的句式，复用率高
2. 工具型内容（合集/商店/仓库）送达率稳定在 2.5%+，适合作为流量底盘
3. 「最强/白嫖/穷鬼」等极限词 + 工具名，是低成本高打开率的组合
4. 具体人名 + 冲突场景（刘润和罗振宇吵架）是好奇心触发的最强形式

**不做什么：**
1. 不要只拆解别人的内容（Dan Koe、播客），要把别人的内容变成「我的实操」
2. 不要用英文词汇堆标题（Is All You Need 这类）
3. 不要写纯观点/纯思考类，没有具体工具/场景/结果的文章打开率低
4. 朋友圈体（无标题）不适合公众号，送达率极低

**数据基准线（用于自我评估）：**
- 低于 1.63%（中位数）：需要反思标题和选题
- 1.63% ~ 3.04%：正常水平
- 超过 3.04%（Top20 均值）：爆款区间
- 超过 5%：现阶段天花板级别
