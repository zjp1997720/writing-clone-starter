## 结构公式

**失败开场 → 问题诊断 → 寻找方案 → 疯狂想法 → 三步执行 → Before/After → 意义升华 → CTA**

### 详细拆解

| 阶段 | 作用 | 字数占比 |
|------|------|----------|
| 失败开场 | 制造共鸣，降低防备 | 10% |
| 问题诊断 | 引入外部视角，增加可信度 | 10% |
| 寻找方案 | 铺垫权威人物，解释「为什么选他」 | 15% |
| 疯狂想法 | 抛出核心概念，制造好奇 | 10% |
| 三步执行 | 降低门槛，让人觉得可复制 | 15% |
| Before/After | 视觉冲击，效果不用解释 | 15% |
| 意义升华 | 从个案升维到方法论 | 15% |
| CTA | 引导行动 | 10% |

## 为什么好

1. **开场不吹牛**：先承认失败，读者放下防备
2. **问题外部化**：让朋友说出问题，比自己说更可信
3. **权威背书**：薛辉的身份为后续内容增加可信度
4. **三步法**：任何复杂流程拆成三步，都显得可操作
5. **Before/After**：视觉对比胜过千言万语
6. **意义升华**：把「教程」升级为「认知」，读者觉得学到了

## 4D标签

- 哲学层：最后升华到「学习本质」
- 心理层：失败共鸣 + 即时满足（三步法）
- 传播层：经典的「问题-方案-效果」结构变体
- 社会层：暗示「会用 Skill 的人」是少数先行者
