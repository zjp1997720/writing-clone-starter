## 0. 选题方法论 (Topic Selection)

### 事件 > 情绪 > 文案

选题的核心公式：

```
事件 > 情绪 > 文案
让视频在工作中产生
```

**操作方法**:

- 当时发生什么事，当时就拿手机拍
- 不要等回家再想选题
- 事件发生的那一刻，情绪是最真实的

### Loser共鸣法

> "普通人的loser，就是你的流量食物。"

**核心逻辑**:

- 悲观的想法引起大部分悲观人的共鸣
- Loser拍Loser本身就是破局之道
- 丑逼乌托邦：loser自嘲 → 吸引loser → 共鸣 → 起号

**示例**:

```
"我的生活太平淡了，我是不是就这样庸庸碌碌过一生了？"
→ 这本身就是百万播放级选题

"我想做个副业做自媒体，但我觉得我真的是loser，我没法做。"
→ 这又是一个选题
```

### 即时记录法

薛辉的做法：

- 每天写800字深度思考
- 发生事情立刻记在备忘录
- 刘思毅每天写15条朋友圈

> "你觉得生活平淡，是因为你忘了那些不平凡的瞬间。"

**具体场景**:

- 坐地铁看到漂亮姑娘不敢搭讪 → 记下来，拍出来
- 和客户聊完有感触 → 立刻记录
- 工作中遇到问题 → 当场拍摄

## 0.5 两层漏斗法 (Two-Layer Funnel)

### 适用场景

**行业火，品类不火**的情况：

- 建筑行业火，但卖螺母的不火
- 艺术培训火，但教狐狸舞的不火

### 操作方法

将内容分成两层：

| 层级     | 占比 | 内容     | 目的     |
| -------- | ---- | -------- | -------- |
| 上层漏斗 | 60%  | 行业话题 | 吸引人群 |
| 下层漏斗 | 40%  | 产品内容 | 转化成交 |

### 示例：卖螺母的工厂

**上层漏斗（60%）**: 讲行业话题

```
"现在还在跑工地干工程的，结账比吃屎都难。"
→ 击中所有建筑行业的人
→ 行业人关注你
```

**下层漏斗（40%）**: 讲产品

```
在工厂里做检测、做生产、抓发货
做一个工厂里的质检员形象
→ 用户看到你一丝不苟
→ 信任建立
```

### 为什么要分两层

- 大部分好做的行业一步就能赚钱
- 极端垂直的行业需要两步：先圈人，再转化
- 不要指望一条视频既吸引人又转化

## 1. 黄金三秒法则 (Golden 3 Seconds)

### 核心公式

```
视觉冲击 + 痛点问题 = 留住用户
```

前3秒的唯一任务：**阻止用户划走**。

### ❌ 禁止开头（直接判死刑）

| 错误开头          | 为什么死          |
| ----------------- | ----------------- |
| "大家好，我是xxx" | 没人关心你是谁    |
| "今天给大家分享"  | 废话，直接划走    |
| "最近很多人问我"  | 和我有什么关系？  |
| 慢镜头开场        | 3秒都没说话，走了 |

### ✅ 爆款开头模板

#### 类型A：制造冲突

```
"停！往下滑之前先听我说完。"
"千万别xxx，否则你会后悔一辈子。"
"我今天必须曝光这个行业黑幕。"
```

#### 类型B：痛点反问

```
"你是不是也遇到过xxx？"
"为什么你努力了这么久还是xxx？"
"99%的人都不知道这个秘密。"
```

#### 类型C：惊人结论

```
"我靠这个方法，3个月赚了xxx。"
"这个东西，救了我一命。"
"看完这条视频，你会感谢我的。"
```

#### 类型D：悬念设置

```
"接下来我要说的，可能会得罪很多人。"
"这件事我犹豫了很久，今天终于决定说出来。"
"看到最后，有惊喜。"
```

## 2. 情绪曲线 (Emotion Curves)

短视频的本质是**情绪的传递**。没有情绪的视频，就是流水账。

### 四大情绪驱动

| 情绪    | 触发点     | 应用场景     | 示例             |
| ------- | ---------- | ------------ | ---------------- |
| 😂 快乐 | 反转、搞笑 | 剧情号、vlog | 结局出人意料     |
| 😡 愤怒 | 揭秘、吐槽 | 测评、职场   | 曝光行业黑幕     |
| 😢 共鸣 | 故事、励志 | 情感、成长   | 逆袭故事         |
| 😨 恐惧 | 警告、避坑 | 健康、理财   | "再不注意就晚了" |

### 情绪节奏设计

```
0-3秒：情绪触发点（必须有）
3-15秒：情绪升级
15秒-结尾：情绪释放 + CTA
```

## 3. 脚本结构模板

### 结构A：教程/干货类

```
【0-3秒】Hook：抛出痛点问题
"你知道为什么你的视频没人看吗？"

【3-15秒】价值输出：高密度干货
"第一，xxx；第二，xxx；第三，xxx"

【15秒-结尾】CTA：引导互动
"觉得有用就点个关注，下期教你xxx"
```

### 结构B：故事/剧情类

```
【0-3秒】冲突设置
"那天老板把我叫进办公室，说了一句话，我当场辞职。"

【3-20秒】情节发展
按时间线/因果链展开

【20秒-结尾】反转 + 价值点
意外结局 + "所以记住，xxx"
```

### 结构C：测评/种草类

```
【0-3秒】结论先行
"这个东西，我后悔买晚了。"

【3-20秒】使用体验
痛点 → 解决 → 效果

【20秒-结尾】购买引导
"链接在小黄车，自己去看。"
```

### 结构D：口播/观点类

```
【0-3秒】争议观点
"我说一句可能得罪人的话：xxx"

【3-25秒】论证观点
摆事实、讲道理、举例子

【25秒-结尾】总结 + 互动
"你们怎么看？评论区告诉我。"
```

## 6. 口播适配规则（硬约束）

> 脚本是写给嘴念的，不是写给眼睛看的。写完必须默念一遍，嘴上过不去的地方全部改掉。

### 规则一：断长句

一句话超过 20 个字，就要拆。口播一口气最多说 15-20 个字，超了观众跟不上，你自己也喘不上气。

```
❌ 几乎没见过一个零基础小白，通过「学AI编程→做产品」这条路走通的。
✅ 零基础小白，通过「学AI编程、做产品」这条路走通的，我一个都没见过。
```

```
❌ 产品是你卖的东西，渠道是你怎么触达客户，客户是最终付钱的人。
✅ 产品是你卖啥。渠道是你怎么让客户知道你。客户是最后掏钱的人。
```

**判断方法**：一句话里超过 2 个逗号，必须拆成短句。

### 规则二：去书面词

口播不是写论文。书面词换成嘴上能说出来的词。

| 书面词 | 口播替换 |
|--------|----------|
| 稀缺性的误判 | 「什么东西稀缺」的判断错了 |
| 实现能力不再稀缺 | 会做这件事，不再稀缺 |
| 编排能力，比实现能力值钱 | 会编排，比会实现值钱 |
| 恰恰是AI最擅长的 | AI太擅长了 |
| 愿意持续付费 | 愿意一直买你的东西 |
| 认知比你低两到三个层级的人 | 比你懂得少很多的人 |
| 需求端和供给端同时成熟 | 需求和供给，同时成熟 |
| AI能做什么、不能做什么 | AI能干啥、不能干啥 |
| 从零创造 | 从零开始造 |
| 核查事实 | 查事实 |

**判断方法**：这个词你跟朋友吃饭聊天会说吗？不会就换。

### 规则三：去符号和箭头

脚本里的 `→` `—` `/` 等符号，嘴上念不出来。必须换成口语过渡词。

```
❌ 服务客户→产生素材→做成内容→吸引新客户→服务更多客户
✅ 服务客户，产生素材。素材做成内容，吸引新客户。新客户又变成新素材。
```

```
❌ 学AI编程→做产品
✅ 学AI编程、做产品
```

### 规则四：砍冗余修饰

口播节奏要快。「的」「了」「其实」「本质上」这些词能砍就砍。

```
❌ 看着屏幕上几个AI同时在跑的时候，我脑子里就一个念头
✅ 看着屏幕上几个AI同时在跑，我就一个念头

❌ 你省的是「写」的时间，「编排」的时间一点没省
✅ 写的时间是省了，但编排的时间一点没省

❌ 本质很简单
✅ 意思很简单
```

### 规则五：并列不超过三项

口播里并列超过三项，观众记不住，念的人也容易绕。超过三项就拆成多句。

```
❌ 他们不懂AI，不懂编程，只知道自己不会拍视频、不会写脚本，需要专家手把手教。
✅ 他们也不懂AI。就是不会拍视频、不会写脚本，想找个专家手把手教。
```

### 规则六：自问自答拆长段

干货段落太长，用「自问自答」拆开，给观众喘气的节奏。

```
❌ 第一个，论文排版Skill。解决的是科研人员被期刊格式折磨的痛点。来买的是谁？
✅ 第一个，论文排版Skill。干嘛的？帮科研人员搞定期刊格式。来买的都是谁？
```

### 口播自检清单

写完脚本后，逐条过一遍：

- [ ] 每句话不超过 20 个字？
- [ ] 没有书面词？（朋友聊天会这么说吗？）
- [ ] 没有 `→` `/` `——` 等念不出来的符号？
- [ ] 并列项不超过 3 个？
- [ ] 没有多余的「的」「了」「其实」？
- [ ] 默念一遍，嘴上顺不顺？

## 7. 视频描述与标签（必须附在脚本末尾）

> 每条脚本写完，必须在末尾附上视频描述和 #标签，让用户可以直接复制到发布页面。

### 描述结构

```
## 视频描述

[钩子句：1-2句，把视频最炸裂的点浓缩成独立可传播的短文案]

[行动句：预告下一条 / 引导评论 / 给一个悬念]

#标签1 #标签2 #标签3 #标签4 #标签5 #标签6 #标签7
```

### 描述写法规则

1. **钩子句**：把脚本里最有冲击力的数据或结论提出来，不超过2句话。要让人光看描述就想点开视频。
2. **行动句**：引导下一步动作。可以是预告下一条、引导评论、或用一个问题勾互动。
3. **总字数不超过100字**：描述太长没人看。

### 标签策略

每条视频固定 **7个标签**，按以下结构配：

| 类型 | 数量 | 作用 | 示例 |
|------|------|------|------|
| 精准标签 | 3个 | 匹配视频具体主题 | #AI编程 #Skill开发 #爆款方法论 |
| 泛流量标签 | 3个 | 触达更大人群 | #一人公司 #超级个体 #AI创业 |
| IP标签 | 1个 | 品牌沉淀，系列串联 | #智见AI |

**IP标签必须每条都带**，让用户通过标签能找到你的所有视频。

### 示例

```
## 视频描述

花了几千块学AI编程，课程里教的工具一个月就变样了。
我观察了三件事，彻底想通了：会做这件事不再稀缺，知道该做什么才稀缺。

系列第1条，后面几条更炸裂👇

#AI编程 #一人公司 #超级个体 #AI创业 #Skill开发 #智见AI #长期主义
```

## 8. 视频号口播规则（大鹏专用）

> 视频号脚本不是"写给人看"，而是"说给人听"。必须从文案思维切换到对话思维。

### 核心差异：文案思维 vs 对话思维

| 维度 | 抖音/快手（文案思维） | 视频号（对话思维） |
|------|----------------------|-------------------|
| 目标 | 让人"看" | 让人"听" |
| 句式 | 短句堆叠，断句明确 | 长句连贯，用连接词串联 |
| 语气词 | 几乎没有 | 密集使用："呢""吧""一下""去""都" |
| 称呼 | "你" | "大家""我们" |
| 结构 | 列表式（- 标题：...） | 连贯叙述式（"分别提取出来：标题也就是..."） |

### 规则一：反问式开场 + 具体场景

**不用抽象概念，用具体行为细节**

```
❌ 错误：你收藏夹里躺着300篇爆款文章
✅ 正确：你是不是经常看到爆款的文章，就会转发给自己的文件传输助手？
```

**公式**：`你是不是 + 具体行为 + 反问？`

### 规则二：语气词密集化

每句话至少1个语气词：「呢」「吧」「一下」「去」「都」「就」「也就是」

```
❌ 错误：Dan Koe 一个人运营多个账号
✅ 正确：dankoe他一个人运营了多个自媒体账号

❌ 错误：他从来不从零开始
✅ 正确：但是他从来都不是从零去开始写作
```

**常用语气词清单**：
- 动词前加「去」：去开始、去提取、去进行
- 句中加「呢」：这篇文章呢、然后呢
- 句尾加「吧」：可以试试吧、应该是这样吧
- 加强语气用「都」：从来都不是、全都是

### 规则三：长句连贯（用连接词）

用「但是」「然后」「所以」「因为」串联句子，不要短句堆叠。

```
❌ 错误：他从来不从零开始。因为他收藏的不是成品。而是零件。
✅ 正确：但是他从来都不是从零去开始写作，因为他收藏的不是成品，而是这些可以复用的素材。
```

### 规则四：技术栈前置

演示前必须说明技术栈（Obsidian + Claude Code）。

```
❌ 错误：我输入 `/content-goldmine`
✅ 正确：我们现在打开我基于 Obsidian 和 Claude Code 构建的这个分身系统，然后输入斜杠 content goldmine 这个指令
```

**公式**：`我们现在打开 + 技术栈 + 然后 + 操作`

### 规则五：列表改叙述

不用列表式（- 标题：...），改用连贯叙述。

```
❌ 错误：
它自动提取出：
- 标题：怎么写能一眼抓人
- 开头：前三句用了什么钩子

✅ 正确：
分别提取出来：标题也就是怎么写能一眼抓住人；第二个是开头，前三句话用了什么钩子？然后是金句，哪些句子是可以单独传播的？最后是结构，整篇文章它的叙事公式是什么？
```

**公式**：`分别提取出来：第一个是...也就是...；第二个是...；然后是...；最后是...`

### 规则六：转化更柔和

不用祈使句，改用「如果...可以...」。

```
❌ 错误：想要这套系统，关注公众号...
✅ 正确：如果你对这套系统感兴趣，可以关注我的公众号...
```

### 规则七：称呼用"大家"或"我们"

视频号面向群体，不用"你"。

```
❌ 错误：我现在演示给你看
✅ 正确：我现在给大家演示一下
```

### 视频号口播自检清单

写完脚本后，逐条过一遍：

- [ ] 开场用了反问 + 具体场景？（不是抽象概念）
- [ ] 每句话都有语气词？（呢/吧/一下/去/都）
- [ ] 用连接词串联句子？（但是/然后/所以/因为）
- [ ] 演示前说明了技术栈？（Obsidian + Claude Code）
- [ ] 没有列表式，改用连贯叙述？
- [ ] 转化用了"如果...可以..."？（不是祈使句）
- [ ] 称呼用"大家"或"我们"？（不是"你"）
- [ ] 默念一遍，听起来像在跟朋友聊天？

## 薛辉提醒

- **不要自嗨**：你觉得好笑，观众不一定笑
- **不要铺垫太长**：3秒内必须出钩子
- **不要忘记CTA**：每条视频都是成交机会
- **一具体就深刻**：不要说"我赚了很多钱"，要说"我3个月赚了50万"
- **洞察 > 框架**：用户不想听"一百位大咖对谈"，想听"脚气膏一年能挣五千万"
