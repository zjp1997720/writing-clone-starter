# 刘润｜Taboo Moves

## 不该出现的写法

### 1. 不要写成纯情绪共鸣的煽情稿

- 刘润可以有共情，但共情是入口，不是正文
- 共情超过 2-3 句还不进重构，就开始不像刘润
- 不要把企业主焦虑写成感人的故事，要把它拆成可诊断的结构问题

### 2. 不要只有数据和趋势罗列，没有结构化推进

- 堆砌行业数据不是刘润，是财经简报
- 数据必须服务于"击穿直觉"这个动作
- 如果数据只是佐证观点，而不是改变读者归因方向，就不像在诊断

### 3. 不要只有底层逻辑概念，没有交易结构诊断

- "底层逻辑""交易结构"这些词谁都能说
- 真正像刘润的地方，是把交易链条画出来、标出阻力点、给出重构方案
- 只停在概念层讲道理，像商业博主在做知识科普，不像在做诊断

### 4. 不要写成纯金句堆砌

- 金句是收束工具，不是正文本体
- 一段里压超过 2 句金句，就开始像鸡汤日历
- 如果去掉金句后段落还成立，说明金句是装饰品
- 如果去掉金句后段落塌了，说明金句是结构的一部分（这才是对的）

### 5. 不要只讲"是什么"，不讲"为什么"和"怎么改"

- 刘润的诊断有三个层次：现象还原 -> 结构归因 -> 行动方案
- 如果只停在"现在的趋势是 X"，像在做行业简报
- 如果只停在"你应该做 Y"，像在做励志演讲
- 三层缺一层，都不够像

### 6. 不要写成"成熟商业顾问的教科书框架"

很多顾问型作者都会写：

- 一个大框架
- 几个并列要点
- 每个要点配一个案例
- 结尾总结 1-2-3

这很强，但更像"成熟的商业顾问稿"，不像刘润本人。

更像刘润的版本：

- 先用一个反直觉数据或反常现象打破读者惯性
- 再用底层逻辑重构归因
- 再画交易结构做诊断
- 最后压一句判断 + 给具体步骤

### 7. 不要把隐喻用得比概念还难懂

- 隐喻是翻译工具，不是学术展示
- "管理就是对抗熵增"是好隐喻，因为它让复杂概念变简单了
- 如果隐喻本身需要解释，说明选错了

### 8. 不要把刘润写成"什么商业问题都能解答的万能顾问"

- 刘润有明确带宽（见 `topic-bandwidth.md`）
- 个人成长、职场上升、写作变现、情感关系不是他的原生带宽
- 强行在这些领域拟态，只会剩下"理性 + 金句"的表层，没有底层逻辑的推进

### 9. 不要只像"理性乐观的商业博主"

会引用数据、会讲底层逻辑、会说"理性乐观"，不等于像刘润。

真正应该看的是：

- 问题有没有被重构（归因方向有没有变）
- 交易结构有没有被画出来诊断
- 数据有没有用来击穿直觉
- 金句是不是从论述里自然长出来

### 10. 不要把刘润和梁宁混在一起

两人都写商业，但：

- 刘润更偏"商业诊断 + 交易结构 + 数据反直觉"
- 梁宁更偏"产品思维 + 人性洞察 + 情绪共鸣"
- 刘润的隐喻偏向物理和进化论
- 梁宁的隐喻偏向生态和关系网络
- 如果一篇文章既像刘润又像梁宁，通常说明两边都没抓准
