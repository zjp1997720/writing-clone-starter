# 刘润｜Acceptance Checklist

## 用法

这份清单只用于验收刘润高拟态正文。

默认只看 `response.md`，把 `run_summary.md` 当辅助证据，而不是主证据。

## 一、标题 / 开头

- [ ] 标题或开头是否包含反直觉元素或反常数据
- [ ] 是否在 2-3 句内从共情进入问题重构
- [ ] 问题归因方向是否被改变（从外部归因改为内部/结构归因）
- [ ] 是否能让企业主读者迅速产生"这说的是我"的感觉
- [ ] 开头是否避免了空洞的行业背景铺垫

## 二、正文推进

- [ ] 是否至少完成一次"数据击穿直觉"的动作
- [ ] 是否至少完成一次"交易链诊断"（画出链条、标出阻力点、给重构方案）
- [ ] 是否至少完成一次"概念 -> 隐喻翻译"让复杂概念变简单
- [ ] 是否至少完成一次"反常数据 -> 品类属性变迁"的洞察推导
- [ ] 正文是否避免了"大框架 + 并列要点 + 案例配对"的教科书模板
- [ ] 是否至少完成一次问题重构（归因方向改变）

## 三、像度判断

- [ ] 像度是否主要来自动作链（共情 -> 重构 -> 击穿 -> 诊断 -> 收束），而不是术语和金句
- [ ] 是否避免了"理性商业顾问标准课"的表层 cosplay
- [ ] 不看 `run_summary.md`，只看正文，是否仍能感到刘润的推进方式
- [ ] 是否避免了与梁宁的推进方式混淆（人物故事主导 vs 数据击穿主导）
- [ ] 金句是否从论述里自然长出来，不是贴上去的装饰

## 四、结尾

- [ ] 结尾是否压成了一句可传播的判断
- [ ] 是否同时给了具体到"今晚就能做"的行动步骤
- [ ] 是否避免了空洞励志和 checklist 总结
- [ ] 判断和步骤之间是否有逻辑接续

## 五、最终通过标准

至少满足下面三条，才算刘润高拟态通过：

1. 正文中能看到一条完整动作链：共情 -> 重构 -> 数据击穿/交易诊断 -> 金句收束 -> 具体步骤
2. 差异主要落在 `response.md`，而不是靠说明层解释
3. 抹掉术语和金句后，正文仍有刘润式的问题重构和结构诊断感

## 六、90+ 额外门槛

如果命中下面任一条，默认不判 90+：

1. 中段连续 2 段以上在解释框架/概念，而不是通过数据击穿或交易诊断推进
2. 数据引用了但没有解释"为什么感受和数据不一致"
3. 交易结构只在概念层提及，没有画出链条和标出阻力点
4. 金句超过 4 句，开始稀释为鸡汤密度
5. 整体更像"成熟商业顾问的标准课"，而不是刘润在做诊断
6. 与梁宁的推进方式混淆，人物故事和情感共鸣占比过高
