## 计划 held-out 分类

等素材补齐后，按以下维度分配：

### A. 账号诊断 / 流量分析

- 典型"先要数据再下诊断"的场景
- 验证：诊断链是否稳定复现

### B. 变现路径 / 商业倒推

- 典型"先问变现再谈流量"的场景
- 验证：倒推逻辑是否在不同行业话题中稳定出现

### C. 脚本工程 / 内容创作

- 典型"极端案例→通用原理"的场景
- 验证：案例论证结构是否稳定

### D. 对标拆解 / 人设定位

- 典型"否定自己想→市场验证"的场景
- 验证：逆向思维是否在不同场景中复现

## 当前盲测问题

等 held-out 集就位后，用以下问题验证：

1. 不看说明层，只看标题和开头，是否能看出薛辉式的变现倒推入口？
