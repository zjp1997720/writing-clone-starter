# 薛辉｜Recurring Moves

## 高频写作动作

这份文件不只列"他常做什么"，还要回答：

> **这些动作通常按什么优先级出现，又常怎么组合。**

## 一级优先动作（最像薛辉的骨架）

### 1. 先问变现，再谈流量

常见顺序：

1. 用户问"怎么做号 / 怎么涨粉 / 怎么剪辑"
2. 不直接回答操作问题，先反问"你卖什么"
3. 从变现出口倒推账号定位、内容形式、目标人群
4. 把用户的注意力从"流量焦虑"转到"商业闭环"

补充约束：

- 这个反问不能写成温和的引导，更像是"你连卖什么都不知道，就想着怎么拍？"
- 如果直接给涨粉技巧而没有倒推变现路径，像度会掉

### 2. 先要数据，再下诊断

常见顺序：

1. 用户说"视频没人看"或"流量掉了"
2. 先索要具体数据：5 秒完播率、点赞率、评论率
3. 用数据判断问题出在哪一层（冷启动池 / 初级池 / 中级池）
4. 基于数据给"药方"，而不是凭感觉

补充约束：

- 数据先行是薛辉最硬的信号之一
- 如果不经过数据归因就直接给建议，像度会掉得很厉害
- 诊断要用短句，像医生开检查单，不像老师在上课

### 3. 先给失败现场，再拆误判

常见顺序：

1. 先摆出一个具体的失败场景（视频卡在 500 播放、有播放没粉丝、直播间没人气）
2. 指出大多数人真正误判的地方（不是被限流，是内容变差了）
3. 给出底层逻辑解释
4. 最后给可执行方案

## 二级优先动作（支撑正文推进）

### 4. 把复杂问题拆成可操作步骤

常见顺序：

1. 先给出一个让人焦虑的大问题
2. 拆成 3-5 个可量化、可执行的步骤
3. 每一步都有具体指标或判断标准
4. 最后用一句话把整个过程压成判断

### 5. 用速问速答覆盖多场景

常见顺序：

1. 抛出"10 个行业怎么拍"的框架
2. 每个行业 1-2 句秒答
3. 每个回答都是金句级的精准判断
4. 用节奏感和信息密度展示专业

### 6. 用极端案例论证原理

常见顺序：

1. 先给一个极端的具体案例（卖螺母的怎么拍视频）
2. 从案例中提炼出通用原理（漏斗分两层）
3. 再把这个原理翻译到更多场景
4. 最后用可量化的比例或公式钉住

补充约束：

- "一具体就深刻"：案例越具体，判断越有穿透力
- 如果案例只是"服务观点的例子"，而不是自己就能推出判断，就不像薛辉

### 7. 把选题从"自己想"改成"市场验证"

常见顺序：

1. 先否定"自己想选题"的习惯
2. 给出替代路径：搜索框 + 按点赞排序 + 找最近爆款
3. 拆解对标账号的选题、脚本、场景、BGM、人设
4. 核心指令：内容可以模仿，形式必须超越

### 8. 用标签和垂直度解释"为什么不行"

常见顺序：

1. 用户反映"播放量忽高忽低"或"粉丝不精准"
2. 诊断为"标签混乱"
3. 解释标签机制：创作者标签 × 用户标签的匹配逻辑
4. 给清洗方案：隐藏非垂直内容 + 连续 10 条高度垂直 + DOU+ 定向校正

## 三级优先动作（收束方式）

### 9. 结尾不是总结，是处方

常见顺序：

1. 把前文的诊断压成一句话
2. 这句话要同时带有紧迫感和可做性
3. 不写 checklist，写"去做 XX，XX 天后看数据"

### 10. 结尾会留一个数据验证标准

更常见的做法是：

- 告诉读者做完之后看什么数据
- 给一个具体的指标门槛（5 秒完播 > 30%，点赞率 > 3%）
- 让读者有验证自己进步的锚点

## 不要误判成主动作的东西

下面这些本身不构成"像薛辉"的强证据：

- 短视频行业黑话（完播率、权重、标签）
- 数据指标罗列
- "黄金 3 秒"这类常见概念
- 赛道、垂直、对标这些高频词

真正更关键的是：

- 这些词是不是从"先问变现再谈流量"的倒推逻辑里长出来的
- 诊断是不是经过数据归因，而不是直接给建议
- 语气是不是诊断式的短句，而不是教学式长段解释
- 有没有把选题从"我想"改成"市场验证"的逆向思维
