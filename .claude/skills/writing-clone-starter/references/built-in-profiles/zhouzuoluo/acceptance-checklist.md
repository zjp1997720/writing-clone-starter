# 粥左罗｜Acceptance Checklist

## 用法

这份清单只用于验收粥左罗高拟态正文。

默认只看 `response.md`，把 `run_summary.md` 当辅助证据，而不是主证据。

## 一、标题 / 开头

- [ ] 标题或开头是否先给了高识别入口（名人 / 数字 / 身份 / 热点）
- [ ] 是否很快把入口翻译成普通人判断，而不是停在八卦层
- [ ] 是否能让大盘读者迅速产生“这篇和我有关”的感觉
- [ ] 入口是否同时带有现实风险 / 代价感，而不是只有漂亮对比
- [ ] 对交易/第一笔钱类题，入口是否先给出可目击的失败交易场景，而不是泛痛点问句

## 二、正文推进

- [ ] 正文是否至少完成一次“故事 / 案例 -> 观点 -> 金句”的推进
- [ ] 是否至少有一处把小众题材改写成更大盘的公共问题
- [ ] 是否把成长判断落回现实收益、工作、环境、选择或内容变现
- [ ] 小标题或段落安排是否有扩大读者代入面的意识
- [ ] 是否存在一条主故事链，而不是两个并列标准案例平均发力
- [ ] 中后段是否至少完成一次“再抬高一层再压回现实”的转折，而不是均匀平推解释

## 三、像度判断

- [ ] 像度是否主要来自动作链，而不是冒号、数字、热点人物这些表层信号
- [ ] 是否避免了“爆款写作老师腔 + 粥左罗词包”的表层 cosplay
- [ ] 不看 `run_summary.md`，只看正文，是否仍能感到粥左罗的入口和推进方式
- [ ] 是否避免了连续高频复用“很多人不是……而是……”这类公共反转句壳
- [ ] “内容 -> 信任 -> 产品”是否从损失场景里长出来，而不是被作者讲成完整框架

## 四、结尾

- [ ] 结尾是否压成了一句更适合转发的判断
- [ ] 是否同时给了一个现实动作入口，而不是单纯 checklist 总结
- [ ] 最后一击是否带有“现在不做会失去什么”的代价感

## 五、最终通过标准

至少满足下面三条，才算粥左罗高拟态通过：

1. 正文中能看到一条完整动作链：入口翻译 -> 故事推进 -> 判断压实 -> 可传播收束
2. 差异主要落在 `response.md`，而不是靠说明层解释
3. 抹掉几个常见爆款信号以后，正文仍有粥左罗的现实判断和大众翻译感

## 六、90+ 额外门槛

如果命中下面任一条，默认不判 90+：

1. 中段连续 2 段以上在解释市场/方法/框架，而不是通过场景压判断
2. 交易/第一笔钱类题没有明确的失败交易场景入口
3. “内容 -> 信任 -> 产品”主要靠定义成立，不是从损失现场自然长出来
4. 虽然很强，但整体更像成熟写作老师稿，而不是本人稿
