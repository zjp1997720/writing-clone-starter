# Justin Welsh｜Acceptance Checklist

## 用法

这份清单只用于验收 Justin Welsh 高拟态正文。

默认只看 `response.md`，把 `run_summary.md` 当辅助证据，而不是主证据。

## 一、标题 / 开头

- [ ] 标题或开头是否给了第一人称数字锚点或具体痛点场景
- [ ] 是否很快把锚点翻译成可复制路径或直接行动指令
- [ ] 如果用了数字，是否同时给了排除条件（zero employees, no paid ads 等）
- [ ] 开头是否让目标读者（solopreneur / creator）立刻产生"这写的是我"的感觉

## 二、正文推进

- [ ] 正文是否至少完成一次"编号清单 + 经营判断"的推进
- [ ] 是否至少出现一个可量化公式或框架（30-30-30, content matrix, niche formula 等）
- [ ] 是否有 owned-audience 意识（邮件列表、私域资产），且贯穿而非只出现在某一节
- [ ] 反共识判断是否在正文中承担论证功能，而不只是装饰性金句
- [ ] 如果是定位/选题题，是否用 Niche of One 的逻辑拆解（利基 + 问题 + 差异化方案）
- [ ] 正文是否避免了均匀平推的"tip + explanation"清单，而是每条都有判断或取舍

## 三、像度判断

- [ ] 像度是否主要来自动作链（数字锚点 -> 公式 -> 清单 -> 行动指令），而不是编号清单、英文短句这些表层信号
- [ ] 是否避免了 generic business coach 腔 + Justin Welsh 词包的表层 cosplay
- [ ] 不看 run_summary.md，只看正文，是否仍能感到 Justin Welsh 的数字锚点和公式化推进
- [ ] 正文推进方式是否明显不同于 Dan Koe 的哲学框架推进

## 四、结尾

- [ ] 结尾是否给了一个明确的、极简的下一步行动
- [ ] 是否避免了 checklist 式总结和圆满感悟
- [ ] 最后一击是否同时带行动指令和紧迫感

## 五、最终通过标准

至少满足下面三条，才算 Justin Welsh 高拟态通过：

1. 正文中能看到一条完整动作链：数字锚点/痛点 -> 公式化拆解 -> 经营判断 -> 极简行动指令
2. 差异主要落在 `response.md`，而不是靠说明层解释
3. 抹掉编号清单和英文短句这些表层信号后，正文仍有 Justin Welsh 的公式推进和漏斗意识

## 六、90+ 额外门槛

如果命中下面任一条，默认不判 90+：

1. 正文连续 2 段以上在解释概念/方法，而不是通过公式或判断推进
2. 反共识金句只出现在段尾做传播锚点，不在正文中承担论证功能
3. 漏斗/变现链路主要靠定义成立，不是从损失场景自然长出来
4. 虽然很强，但整体更像 generic business coach 稿，而不是 Justin Welsh 本人稿
5. 正文推进方式更像 Dan Koe 的哲学抽象，而不是 Justin Welsh 的公式工程
