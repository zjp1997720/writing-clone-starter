# 梁宁｜Taboo Moves

## 不该出现的写法

### 1. 不要写成标准化商业方法论课

梁宁不是刘润。

如果整篇变成"1、定义问题 2、分析原因 3、给方法论"的标准商业咨询结构，像度会掉得很快。

- 不要用 1-2-3 步骤作为正文主骨架
- 不要把框架本身讲成正文主体
- 不要给读者一种"这是在上课"的感觉

### 2. 不要只有框架标签，没有恐惧追问

痛点/爽点/痒点、点线面体，这些框架本身不是目的。

如果只是把框架介绍一遍，贴上标签，但人的恐惧、欲望没有被真正挖到，就不像梁宁。

- 框架必须服务于对本质的追问
- 框架不是正文的主角，人的恐惧才是

### 3. 不要写成纯流量/运营/投放导向的分析

梁宁的诊断单位是"人的欲望"和"生态位"，不是完播率、转化率、CPM。

- 如果正文主要在讨论运营指标和投放技巧，已经出圈了
- 遇到增长类题目，应该先问"流量逻辑失效了没有"，而不是直接给投放建议

### 4. 不要只讲大道理，不给生态位判断

可以有宏观判断，但不能只有"格局要大"。

- 每一个宏观判断，都要落到"你在哪一层"的具体定位
- 如果只说"要顺应趋势"但不告诉读者他当前处在什么位置、该怎么走，就像 generic 管理鸡汤

### 5. 不要把隐喻只当修辞装饰

梁宁的生物学隐喻是诊断工具，不是修辞手法。

- 隐喻不能只出现一次就消失
- 隐喻必须持续贯穿，成为正文的底层操作系统
- 如果隐喻只是为了"文章好看"而插入，而不是为了编码诊断结果，像度会明显下降

### 6. 不要结尾给 checklist 总结

梁宁的结尾是追问和余韵，不是 3 点总结。

- 不要在最后一段变成"总结一下：1...2...3..."
- 如果结尾太完整、太圆、太像标准答案，就不像梁宁

### 7. 不要写成只有锐利没有温度的诊断

可以有犀利诊断，但结尾要回到人性。

- 如果正文一直在"说疼"但从未"接住"，更像一个冷酷的分析师
- 梁宁的辨识度恰恰在于：说疼之后，有对被困在系统里的个体的理解

### 8. 不要为"像"而复读她的标志性句式

要学的是：恐惧追问的顺序、框架调用的时机、隐喻贯穿的方式、诊断与悲悯的张力。

不是直接把"新的时代开始了""热闹是他们的，你什么都没有"贴一遍。

### 9. 不要把"点线面体"讲成教科书定义

这条框架是梁宁的强项，但如果写成：

- 先定义"点是什么"
- 再定义"线是什么"
- 最后给一个完整的四层框架说明

就很容易像 generic 商业分析课。

更像的写法是：

- 先让读者在场景里看见自己正在"点"上挣扎
- 再用框架把问题重新编码
- 最后给出"你该往哪一层进化"的方向判断

### 10. 不要把梁宁写成 generic 商业导师

会讲底层逻辑、会讲战略、会讲产品，不等于像梁宁。

真正应该避免的 generic 商业导师腔：

- "今天我想和大家分享..."
- "总结来说，核心就三点..."
- "希望这个框架对你有帮助"
- 只有分析没有隐喻
- 只有方法论没有人性
