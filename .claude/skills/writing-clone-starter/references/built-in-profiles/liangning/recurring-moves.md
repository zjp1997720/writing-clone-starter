# 梁宁｜Recurring Moves

## 高频写作动作

这份文件不只列"她常做什么"，还要回答：

> **这些动作通常按什么优先级出现，又常怎么组合。**

## 一级优先动作（最像梁宁的骨架）

### 1. 先追问恐惧/欲望的本质，再给框架定位

常见顺序：

1. 先把表面的商业现象还原成人的底层恐惧或欲望
2. 再用痛点/爽点/痒点或点线面体做框架定位
3. 让读者觉得"原来问题的根在这儿"

补充约束：

- 不能只用框架贴标签
- 更像梁宁的版本，是框架服务于对恐惧的追问，而不是框架本身成为正文主角
- 如果框架只是被介绍了一遍，但恐惧没有被真正挖到，就像 generic 商业导师稿

### 2. 用生物学/地理隐喻做诊断编码

常见顺序：

1. 先用日常语言描述商业问题
2. 迅速切入生物学隐喻（进化、生态位、丛林、基因、物种）
3. 或切入地理隐喻（大河流向、版图、悬崖、冻土层、草原）
4. 用隐喻把问题重新编码，让读者"看到"结构

补充约束：

- 隐喻不是为了修辞好看，而是为了把复杂系统压缩成可感知的画面
- 如果隐喻只是装饰性出现一次就消失，像度会下降
- 更像梁宁的版本，隐喻会持续贯穿，成为正文的底层操作系统

### 3. 把个体/产品放进更大的系统里定位

常见顺序：

1. 先看用户当前的问题（一个产品、一个瓶颈、一个选择）
2. 抬高一层：这个问题在更大系统里的位置是什么
3. 再追问：你是在"点"上挣扎，还是已经在"线"上了

补充约束：

- 不是泛泛的"格局要大"
- 而是精确的生态位定位：你在哪一层，你的生存策略应该是什么
- 如果只说"格局要大"但不给层级判断，就像 generic 管理鸡汤

## 二级优先动作（支撑正文推进）

### 4. 一段通常是"诊断场景 -> 框架调用 -> 隐喻重编码"

常见顺序：

1. 先给一个具体的产品/商业/用户场景
2. 调用痛点/爽点/痒点或点线面体做诊断
3. 用隐喻把诊断结果重新编码成可感知的画面

这让她的正文既有结构，又有画面感。

补充约束：

- 与其用多个并列案例平均推进，不如用一个诊断场景扛住压力
- 场景里要有：具体产品/用户处境、底层恐惧、生态位判断
- 如果场景只是"说明观点的例子"，而不是自己就能推出诊断，正文会很像 generic 商业分析

### 5. 穿插历史纵深和人文回响

常见顺序：

1. 在分析当下问题时，拉一条时间线（互联网发展史、中国古代史、工业革命）
2. 让当下的问题变成一个更长的故事中的一环
3. 增加说服力的同时，给正文增加厚度

补充约束：

- 历史引用不是掉书袋，而是服务于诊断
- 如果引用了历史但没有回扣到当下问题，就像学术文章不像梁宁

### 6. 诊断犀利，结尾留温度

常见顺序：

1. 正文中诊断可以很直接、很锐利
2. 但在收尾时，回到对人性的理解和悲悯
3. 让读者既被"说疼了"，又被"接住了"

补充约束：

- 温度不能是空泛的鼓励
- 更像梁宁的版本，是在指出系统性问题之后，对被困在系统里的个体表达理解

### 7. 先指出"你还在用旧逻辑看世界"，再给新框架

常见顺序：

1. 先指出用户/行业还在用旧时代的逻辑
2. 再给出新时代的框架和判断
3. 制造认知断裂，然后重建

如果缺了"旧逻辑失效"这一步，正文容易像在讲框架课，不像在做诊断。

## 三级优先动作（收束方式）

### 8. 结尾不是总结，而是追问本质或回到人性

常见顺序：

1. 先把前文的诊断压回一个更大的本质问题
2. 再回到人的恐惧、欲望、孤独或梦想
3. 留下哲学余韵，而不是 checklist 总结

补充约束：

- 收尾不能写成"3 点总结"
- 更像梁宁的收尾，往往是一个追问或一个回扣人性的判断
- 如果结尾太完整、太圆、太像答案，像度反而会掉

### 9. 结尾可能留一个生态位判断或方向性建议

更常见的做法是：

- 不给具体操作步骤，给方向性判断
- 告诉读者应该往哪个层级进化
- 把更具体的路径留给读者自己去走

## 不要误判成主动作的东西

下面这些本身不构成"像梁宁"的强证据：

- "底层逻辑"这个词
- "生态位"这个词
- 痛点/爽点/痒点这些框架名称
- 任何商业导师都会说的"格局""战略"

真正更关键的是：

- 框架是不是服务于对恐惧的追问
- 隐喻是不是持续贯穿而不是装饰性出现
- 是否有犀利诊断和人文悲悯的张力
- 是否把个体放进了更大的系统里定位
- 是否避免了 generic 商业导师的 1-2-3 方法清单
