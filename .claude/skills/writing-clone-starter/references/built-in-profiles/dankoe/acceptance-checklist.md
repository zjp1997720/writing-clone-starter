# Dan Koe｜Acceptance Checklist

## 用法

这份清单只用于验收 Dan Koe 高拟态正文。

默认只看 `response.md`，把 `run_summary.md` 当辅助证据，而不是主证据。

## 一、开头

- [ ] 是否完成了明确的 Why now 进入，而不是空泛趋势句
- [ ] 是否至少完成一次“旧默认失效”的动作
- [ ] 是否建立了新的默认设定或新的解释坐标

## 二、正文推进

- [ ] 是否完成了至少一次真正的重定义，而不是同义换词
- [ ] 是否把一个复杂问题压成了低维骨架、极简公式或结构框架
- [ ] 是否完成了至少一次“战术 -> 身份 / 系统 / 主权 / 长期生活设计”的抬升
- [ ] 隐喻是否承担了结构职责，而不是只负责高级感

## 三、像度判断

- [ ] 像度是否主要来自动作链，而不是 system / map / sovereignty 这类公共信号词
- [ ] 是否避免了“普通强稿 + Dan Koe 名词包”的表层 cosplay
- [ ] 不看 `run_summary.md`，只看正文，是否仍能感到 Dan Koe 的推进节奏

## 四、结尾

- [ ] 结尾是否给的是“起步位置 + 更大方向”，而不是 checklist 总结
- [ ] 读完后留下的是路径感，而不是“教程已经结束”的完成感

## 五、最终通过标准

至少满足下面三条，才算 Dan Koe 高拟态通过：

1. 正文中能看到一条完整动作链：进入 -> 重定义 -> 抬升 -> 收束
2. 差异主要落在 `response.md`，而不是靠说明层解释
3. 抹掉几个招牌词以后，正文仍有 Dan Koe 的结构和推进感
