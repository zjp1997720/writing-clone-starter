# Dan Koe｜Topic Bandwidth

## 核心带宽（优先高拟态）

这些主题最适合直接进入 Dan Koe 高拟态模式：

1. 一人公司 / solopreneur
2. 个人品牌 / 创作者成长
3. 写作系统 / 长文写作 / 内容系统
4. 自我升级 / 身份重塑 / 习惯系统
5. AI 时代的个体杠杆
6. 兴趣、技能、价值、事业的一体化
7. perspective / worldview / intellectual signature

## 邻近带宽（可拟态，但要谨慎）

1. 学习方法
2. 生产力 / 深度工作
3. 数字产品 / 服务化知识
4. 创业起步与高客单价 offer
5. 长期主义与生活设计

这些主题能写，但必须尽量拉回 Dan Koe 常见的解释框架：

- 系统

## 弱带宽（默认降级）

命中这些主题时，默认不建议继续高拟态：

1. 中国时政 / 本地新闻评论
2. 强平台运营细节
3. 企业管理制度细节
4. 重数据调研报告
5. 法律 / 财税 / 政策实操
6. 纯产品 PR 稿
7. 需要大量中国现实案例压实的行业分析

## 降级触发器

如果出现以下任一情况，直接触发降级提醒：

1. 主题必须依赖大量中文现实语境
2. 文章主要价值来自本地案例，而非作者视角
3. 主题更像新闻解释，而非长期认知与身份升级
4. 强行拟态会只剩句式像，世界观和推进已经不像

## 降级呈现规则

### 默认规则

触发降级后：

- 默认把降级原因写进 `run_summary.md`
- 默认把 `response.md` 交付成干净的强文章正文，不附任何降级前置说明
- 不要把“我现在按规则降级”这类 system note 直接写成正文第一句

### 如果用户明确追问为什么没有继续拟态

优先在对话说明或 `run_summary.md` 里解释，而不是写进 `response.md`。

只有在任务明确要求“把降级原因一起给用户展示”时，才允许单独加一个与正文彻底分隔的说明块。

## 内部提醒口径（不要默认进正文）

> 这个主题已经超出 Dan Koe 的原生强项带宽。继续硬拟态，文章会更像表层模仿，不像本人。按 starter 规则应切到强文章模式，并把降级原因留在说明层，而不是默认写进正文。
