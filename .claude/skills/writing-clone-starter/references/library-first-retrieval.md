# 强文章模式｜素材库优先取料

## 目的

把“素材库高权重”从抽象原则变成原型级取料策略。

## 总原则

- 不是所有原型都同样取料
- 不是所有原型都让素材库承担同样多的骨架
- 先定原型，再定取料，不要反过来让素材拖着结构走
- starter 的外部素材默认只来自：`02_素材库/writing-clone-starter-material-library/`

## 1. 观点拆解型

### 优先取什么

- 反常识判断
- 旧默认
- 金句
- 结构型素材
- 机制解释

### 素材承担什么角色

- 主要承担“拆旧默认”和“立新判断”的支撑
- 素材要帮正文推判断，不是帮正文显得见多识广

## 2. 方法教程型

### 优先取什么

- 步骤框架
- 操作清单
- 易错点
- 前后对比
- 小型示例

### 素材承担什么角色

- 主要承担步骤压实与降低执行门槛
- 素材要帮正文变得更可执行，而不是更宏大

## 3. 案例复盘型

### 优先取什么

- 案例经过
- 关键转折
- 结果
- 机制解释
- 对照样本

### 素材承担什么角色

- 主要承担事实链与机制链的连接
- 素材要让“发生了什么”变清楚，也要让“为什么会这样”站得住

## 4. 认知升级型

### 优先取什么

- 概念拆解
- 框架迁移
- 类比
- 认知冲突素材
- 小案例验证

### 素材承担什么角色

- 主要承担新坐标系的解释与压实
- 素材要服务框架成立，不要把正文拖成素材拼贴

## 禁止项

- 不要先拉一堆素材，再临场决定文章骨架
- 不要把素材库高权重理解成“素材越多越好”
- 不要为了显得高级，默认取作者气质型素材去替代原型骨架
