# 一个最简单的 skill，把 Claude Code 打造成数字分身

之前我写过一篇文章，讲怎么用 Claude Code 帮商会写稿子。那篇文章里，我把几百兆的商会资料文件夹权限给了 Claude Code，让它自己翻档案、提取语料、匹配图片，10 分钟就搞定了一篇年会报道。

效果确实好。但用了一段时间，我发现一个问题：**每次开新对话，它又变成了傻大春。**

上次聊的项目背景忘了，孙会长的语料库位置忘了，我偏好的文风也忘了。又得从头交代一遍。你可能会说，不是有 `CLAUDE.md` 吗？把偏好写进去不就行了？

我试过。**但 `CLAUDE.md` 解决不了根本问题。** 它是 **静态的、面状的**。你把一堆信息平铺在那里，每次对话都全部加载进去。

但人的记忆不是这样的。人的记忆是 **线性的、有时间顺序的**。我记得上周跟它聊过文件处理的坑，我记得昨天它发现了一个好用的语料提取方法，我记得今天早上它还在处理一个没做完的任务。这些都是 **有时间线的经验积累**。

如果你想让 Claude Code 真正成为你的「数字分身」，而不只是一个每次都要重新介绍自己的工具，它就得有这种 **线性记忆** 的能力。

废话不多说，今天分享一个我正在用的方案。

## 先说结论

日本开发者 yamadashy 做了一个叫 **agent-memory** 的 Skill，专门解决这个问题。

它做的事情很简单：

- 说「记住这个」，它就把当前的要点保存成文件
- 说「回忆一下」，它就去翻之前保存的记忆
- 每条记忆都有时间戳、摘要、标签，**按时间线积累**

这不是官方出的，但思路非常实用。关键是，它用的是 **Agent Skills 开放标准**。这意味着不只 Claude Code 能用，Cursor、VS Code Copilot、Codex CLI 等支持这个标准的工具都能用。

## 工作原理

### 存储结构

所有记忆存储在 `.claude/skills/agent-memory/memories/` 目录下。不是一个大文件塞所有内容，而是 **按类别分文件夹，按时间线积累**。

每个记忆是一个独立的 Markdown 文件，带有标准化的前置元数据。`summary` 是关键。Claude 会先用 `rg` 搜索所有相关记忆文件的 summary 行，找到相关的再读全文。

这和 Agent Skills 的 **渐进式披露** 是同一个思路：先看摘要判断相关性，需要再读详情。

### 触发机制

不是自动记录所有对话，而是 **你主动触发**：

- 「记住这个」「保存一下」→ 创建记忆
- 「想起来……」「我们之前聊过……」→ 检索记忆
- 「整理记忆」「清理过时的」→ 维护记忆

它也会在发现值得保存的东西时主动询问：「这个调查结果要保存吗？」

## 用在商会秘书场景

回到我的商会秘书案例。以前每次开新对话，我都要说：「商会资料在 /Documents/商会，孙会长语料库在子文件夹里，我们习惯用这种文风……」

现在不用了。第一次配置好之后，我跟它聊完一篇报道，直接说：「记住这个：商会报道的标准流程和孙会长的常用语料。」它自动保存。下次开新对话，我直接说：「帮我写个年会报道，先想起来之前我们是怎么做的。」它自己去翻 memories 目录，找到那条记忆，然后直接开始干活。

不用重复交代，不用每次从零开始。**这才是「数字分身」该有的样子**。它记得你们一起做过什么，记得你的偏好是怎么在一次次协作中形成的。

## 行业趋势：文件系统就是最好的上下文

这个方案能成立，有一个更大的背景：**行业正在朝这个方向走**。

Cursor 刚发布了一篇博客，讲他们的 **动态上下文发现**。核心观点是：

> 与其一开始就把所有信息塞进上下文窗口，不如让 Agent 自己去文件系统里找需要的东西。

他们做了个 A/B 测试，动态上下文发现让 token 使用量降低了 **46.9%**。更重要的是，Cursor 正式支持了 **Agent Skills**。

大鹏这里要说个暴论：2026 年不支持 skill 的都不能称之为 agent。

这意味着什么？**你创建的 Skill，可以跨工具使用。** 今天在 Claude Code 里用的 agent-memory，明天换 Cursor 也能用。

这就是我说的 **降维打击**。重点不在于让每个工具单独记住你的偏好，而在于构建一套 **可迁移的上下文架构**。

## 从工具到方法论

说到这里，你可能发现了，这篇文章其实不只是在讲一个 Skill。它在讲一个更大的问题：**怎么让 AI 真正「懂」你？**

`CLAUDE.md` 是静态的偏好清单。但真正的「懂」，来自 **共同经历**。你跟一个助理合作三年，他懂你，不是因为你给他一份说明书，而是因为他记得你们一起踩过的坑、一起做过的决策、一起形成的默契。

这些记忆是 **有时间线的**。它们串成一条线，构成了「你们一起走过的路」。这个 agent-memory Skill，做的就是这件事——把你和 AI 的协作经历，用结构化的方式沉淀下来。

**Prompt 是一次性的，用完就没了。但 Skill + Memory 是资产，用得越久，价值越大。**

这就是我之前说的：别再学提示词了，要学 **上下文架构**。

## 写在最后

如果你也在用 Claude Code，强烈建议你试试这个方案。配置方法很简单，yamadashy 在他的文章里写得很清楚。用一段时间，你会发现 Claude 越来越「懂」你了。

因为你的知识、你的经验、你的偏好，都在一点点沉淀成它的「第二大脑」。而且这个记忆是 **线性的、有时间顺序的**。就像一个真正的助理，记得你们一起做过什么。
