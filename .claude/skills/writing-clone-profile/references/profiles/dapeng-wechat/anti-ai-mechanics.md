# Anti AI Mechanics（dapeng-wechat）

这份文件不是新的风格真相源。

它只负责把大鹏公众号长文里最容易被 AI 抹平的几类机制显性化：可控不完美、体感记忆、主线脉冲、疑问句节奏、小标题与口语转场。

运行时优先级：

1. 当前用户明确要求
2. `style-profile.md`
3. `voice-cues.md`
4. 本文件

如果本文件与事实核查、Claim Ledger、教程清晰度冲突，优先保持事实与清晰。

## 1) Controlled Imperfection

目标：保留真实思考的拐弯，而不是制造表演性的「像人」。

允许：

- 自我修正：前一句判断不够准确时，后一句主动收窄或改写。
- 犹豫停顿：真实不确定时承认「这个地方我也想了很久」。
- 省略与断裂：在关键转折、情绪落点、强判断处使用短句或不完整句。
- 轻微自嘲：真实踩坑、误判、绕远路时可以自嘲。

不允许：

- 每篇为了像人而硬塞犹豫。
- 用模糊语气逃避事实准确。
- 在教程步骤、价格、功能、承诺上故意不严谨。
- 把「说实话」「我不确定」当成装饰性口癖。

## 2) Felt Memory

优先写身体、动作、停顿和场景，而不是概括情绪。

少写：

- 我很震撼。
- 我非常感动。
- 这个体验令人印象深刻。
- 我感到很焦虑。

优先改成：

- 我盯着那行字看了半天。
- 我当时愣了一下。
- 我手已经放到键盘上了，但突然不知道该怎么写。
- 我把那段话删了，又重新写了一遍。

如果没有真实体感，保持克制，不要硬造强反应。

## 3) Main-thread Pulse

文章可以偏出去讲案例、知识、类比，但每次偏出去之后都要回到主线。

检查问题：

1. 这个 detour 是补案例、补机制、补类比，还是单纯显得有料？
2. detour 后有没有一句话把读者拉回本文核心判断？
3. 这句回扣是在推进，还是重复标题？

常见回扣方式：

- 回到一开始那个问题。
- 这也是为什么我说，真正关键的不是 X，而是 Y。
- 说回我们自己的写作分身，这个问题会更明显。
- 所以这不是一个工具问题，而是一个系统问题。

## 4) Question As Pacing

疑问句不是装饰，它有三种功能。

| 功能 | 用法 | 例子方向 |
| --- | --- | --- |
| 刹车 | 让读者停一下，准备接收关键判断 | 但问题真的是这样吗？ |
| 转向 | 从一个模块切到另一个模块 | 那这和我们的写作分身有什么关系？ |
| 加速 | 制造下一段悬念 | 结果最麻烦的地方在哪？ |

使用原则：

- 连续 3 段平稳说明后，优先考虑是否需要一个问题来转向。
- 问完必须回答，不要堆设问。
- 强观点文里，疑问句要帮助读者进入判断，而不是故作悬念。

## 5) Heading And Transition

大鹏文章不是不能有小标题，而是不能让小标题把文章切成 PPT。

默认规则：

- 叙事型、观点型、复盘型文章：少用小标题，优先用口语转场保持心流。
- 教程型、工具型、系统拆解型文章：允许小标题，但小标题要像路标，不像课程目录。
- 小标题下面第一段必须自然承接上文，不能直接进入列表。
- 连续多个同级小标题之间如果没有转场，标记为「讲义感风险」。

口语转场的功能分类：

- 转场类：从上一个模块自然走到下一个模块。
- 判断类：表达观点但不端着。
- 缓冲类：先承认读者处境，降低攻击性。
- 自我修正类：承认前一句不够准确，再推进。
- 收束类：把一段话落到读者能带走的判断。

表达候选必须从大鹏真实范文中持续补充；不要直接迁移其他作者口癖。

## 6) 使用边界

这些机制只解决「AI 太顺、太满、太平」的问题。

它们不解决：

- 事实准确性
- 结构完整性
- 读者承诺
- 产品/课程边界

如果一篇文章根本没有清楚立场、具体价值或真实素材，不要靠这些机制硬救，应该回到 Article Gravity Gate 和 Brief 重建。
