# Article Archetypes（dapeng-wechat）

这份文件定义大鹏公众号长文的文章级原型。

它不替代 `three-structures.md` 的 A/B/C 开头结构。A/B/C 解决「从哪里切入」，本文解决「整篇文章按什么叙事骨架推进」。

使用顺序：

1. 先根据 Brief 判断文章原型。
2. 再用 `Structure-Eval.md` 判断 A/B/C 哪个入口更适配。
3. 最后用 `Momentum-Map.md` 设计主线、递进、callback。

## 1) 亲历实验 / 跑通闭环型

适用信号：

- 大鹏亲自做了一个工具、工作流、产品、课程、项目。
- 有明确过程：起因、卡点、改法、结果。
- 读者最想带走的是「我也能不能这么做」。

读者承诺：

- 看完知道这个闭环为什么成立，以及自己能从哪一步开始抄作业。

推荐骨架：

1. 用一个真实场景或对话开场，让读者先看到问题。
2. 解释为什么传统做法不够，或者为什么这个痛点值得解决。
3. 展示大鹏实际怎么跑：工具链、步骤、关键决策。
4. 放一个结果快照：产物、数据、截图、交付物、反馈。
5. 抽出可复用方法，而不是只炫耀结果。
6. 结尾落到一个更大的判断：这件事说明了 AI 时代什么能力在变值钱。

节奏曲线：

- 开头要快，用场景抓人。
- 中段可以慢下来拆流程，但每个步骤都要回答「为什么是这一步」。
- 结果段要具体，不要只写「效果很好」。
- 结尾再升一层，但不能突然喊口号。

高质量来源：

- `03_归档/已发布文章/_用于初始化skill_精选代表作/我用Claude_Code___Skill_帮老婆跑通AI_教育的最小变现闭环_AI_古诗课.md`
- `03_归档/已发布文章/_用于初始化skill_精选代表作/我用_Claude_Code_造了个五人智囊团_Team_刘润和罗振宇为我的项目吵了一架.md`
- `01_项目/内容创作/260413_爆改栋哥dbs-skill/草稿_final.md`

常见风险：

- 只讲工具步骤，忘了开头的真实问题。
- 只写结果很牛，不写中间的判断和取舍。
- 销售 CTA 顶得太早，让文章从经验分享变成产品说明。

## 2) 强观点纠偏 / 认知改道型

适用信号：

- 核心是纠正一个流行误解。
- 文章有强判断，例如「真正值钱的不是 X，而是 Y」。
- 读者可能一开始不同意，需要先理解他为什么会这么想。

读者承诺：

- 看完会重新评估一个原本以为理所当然的选择。

推荐骨架：

1. 开头先暴露作者自己的犹豫、误判或亲身代价。
2. 写出流行路径为什么看起来合理。
3. 用 2-3 个观察或案例拆掉这个路径。
4. 把问题收束成「稀缺性」或「边界」判断。
5. 给读者一条更现实的路径。
6. 用一句硬判断收尾，不再重复证明。

节奏曲线：

- 前 300 字必须有冲突或不舒服的真话。
- 中段不要堆 5 个案例，2-3 个足够，后一个要比前一个更靠近核心判断。
- 收尾要短，别写成「以上就是我的建议」。

高质量来源：

- `03_归档/已发布文章/_用于初始化skill_精选代表作/别再劝小白学_AI_编程了_那是杀人诛心.md`
- `03_归档/已发布文章/_用于初始化skill_精选代表作/昨晚，我被 Gemini 骂醒了：技术在这个时代是最廉价的壁垒.md`
- `01_项目/内容创作/260411_AI时代你还在不在场/草稿_final.md`

常见风险：

- 只会下狠话，没有先承认对方为什么会那么想。
- 观点太满，缺少自己的误判和代价。
- 后半段反复证明同一个主张，密度下降。

## 3) 系统拆解 / 工具方法型

适用信号：

- 核心是拆一个 Skill、Agent、工作流、系统架构或方法论。
- 读者需要看懂原理，也需要知道怎么上手。
- 文章含较多术语、步骤或模块。

读者承诺：

- 看完不仅知道它是什么，还知道为什么这样设计，以及自己能怎么迁移。

推荐骨架：

1. 先说一个具体痛点，不从宏大背景开场。
2. 解释现有做法为什么不够。
3. 给出系统的核心结构，最好压成 3-5 个模块。
4. 每个模块都配一个真实使用场景或结果。
5. 抽出底层设计原则。
6. 结尾落到「这类事情以后该怎么做」。

节奏曲线：

- 允许小标题，但小标题要像路标，不要像课件目录。
- 小标题后第一段必须承接上文，不能直接列表。
- 技术细节必须服务主线；读者不需要知道的配置，不要硬塞。

高质量来源：

- `03_归档/已发布文章/_用于初始化skill_精选代表作/手把手教你打造_AI_写作分身_skill_最佳实践.md`
- `03_归档/已发布文章/_用于初始化skill_精选代表作/一个最简单的_skill_把_Claude_Code打造成数字分身.md`
- `01_项目/内容创作/260413_爆改栋哥dbs-skill/草稿_final.md`

常见风险：

- 写成产品文档或安装教程。
- 模块过多，主线变散。
- 用外部概念盖过大鹏自己的实践判断。

## 4) 复盘算账 / 真实经营型

适用信号：

- 文章有收入、阅读、转化、涨粉、课程、商单、私域等真实经营数据。
- 核心不是炫耀结果，而是解释什么动作带来什么后果。
- 读者希望少走弯路。

读者承诺：

- 看完能获得一套更现实的经营判断，而不是被成功学刺激。

推荐骨架：

1. 用一个不舒服的反馈、下滑数据或真实反差开场。
2. 按阶段复盘：发生了什么、做了什么、结果如何。
3. 每一类变现方式都给真实数字或明确边界。
4. 解释失败原因，不只列成功动作。
5. 抽出当前阶段的新策略。
6. 给同行几条少而硬的建议。

节奏曲线：

- 开头要真，不要把复盘写成战报。
- 数据段要具体，但不要为了显得厉害堆数字。
- 建议段要来自前文的坑，不要突然变成通用鸡汤。

高质量来源：

- `01_项目/内容创作/260417_AI自媒体变现复盘/草稿_final.md`
- `03_归档/已发布文章/_用于初始化skill_精选代表作/一人公司怎么做_2025_年我跑出了六位数收入_全网最强拆解.md`

常见风险：

- 只晒收入，不交代代价。
- 把单次数据写成长期规律。
- 在信任修复型文章里过早推产品。

## 5) 概念升维 / 个人困惑型

适用信号：

- 文章起点是一个困惑、恐慌、被打醒的瞬间。
- 核心是把个人体验升维成时代判断或长期原则。
- 读者读完希望获得一种新的自我解释。

读者承诺：

- 看完能说清自己为什么焦虑，以及接下来该守住什么。

推荐骨架：

1. 用一句外部刺激或对话开场，但立刻拉回「这说的是我自己」。
2. 写清作者的具体矛盾：效率提升了，但什么东西丢了。
3. 引入一个框架或概念，但不能让框架成为主角。
4. 用一个强类比把抽象问题变成可感知画面。
5. 回到作者自己的选择或规矩。
6. 结尾给读者一个问题，让他回到自己身上。

节奏曲线：

- 情绪要来自体感和行动，不来自抽象词。
- 类比是中段高潮，不能开头就把牌打完。
- 结尾要克制，但必须有作者自己的下一步。

高质量来源：

- `01_项目/内容创作/260411_AI时代你还在不在场/草稿_final.md`
- `03_归档/已发布文章/_用于初始化skill_精选代表作/昨晚，我被 Gemini 骂醒了：技术在这个时代是最廉价的壁垒.md`

常见风险：

- 过度哲学化，读者看不到自己的关系。
- 只有大词，没有一个具体动作或场景。
- 结尾停在感慨，没有形成规矩或判断。

## 6) 外部高手 / 外部长帖镜子型

适用信号：

- 文章起点是一个外部作者、推文、长帖、方法论或爆文。
- 核心不是介绍对方，而是用对方的内容照见大鹏自己的判断、工具系统或行动问题。
- 读者想要的是「这个外部材料跟我有什么关系」，不是人物百科。

读者承诺：

- 看完能理解外部高手真正有价值的那一层，并知道它如何迁移到自己的工作或生活。

推荐骨架：

1. 用一个外部材料的具体入口开场：某条推文、某篇长帖、某个观点。
2. 先承认标题、概念或对象为什么吸引人，但不要停在介绍。
3. 把外部观点翻译成人话，再接回大鹏自己的经历或系统判断。
4. 用 2-4 个关键点拆解，不做百科式全量总结。
5. 每拆一个点，都回答「它照见了我 / 普通人什么问题」。
6. 结尾落到自己的行动、身份、系统或下一步，而不是替外部作者做总结。

节奏曲线：

- 开头可以借外部材料的势能，但第二屏必须出现大鹏自己的位置。
- 中段要有翻译动作：把外部概念变成读者听得懂、用得上的判断。
- 外部引用不能连续占据两段以上；每次引用后都要回到作者自己。

高质量来源：

- `03_归档/已发布文章/_用于初始化skill_精选代表作/我的数字分身是如何把Dan_Koe爆文讲透的_一天重启人生.md`
- `01_项目/内容创作/260324_别再把AI当聊天机器人而要把它驯化成工作系统/草稿_final.wechat.md`

常见风险：

- 写成外部作者介绍。
- 外部金句太多，大鹏自己的判断太少。
- 只做翻译，不做迁移。

## 7) 路由速查

| Brief 信号 | 优先原型 |
| --- | --- |
| 我亲自试了 / 跑通了 / 做了一个东西 | 亲历实验 / 跑通闭环型 |
| 大多数人都错了 / 不要再 / 真正问题不是 | 强观点纠偏 / 认知改道型 |
| 一个 Skill / Agent / 系统 / 方法论怎么设计 | 系统拆解 / 工具方法型 |
| 收入 / 阅读 / 转化 / 课程 / 私域 / 商单 / 复盘 | 复盘算账 / 真实经营型 |
| 我被一句话打醒 / 我开始恐慌 / 我最近不对劲 | 概念升维 / 个人困惑型 |
| 我看完某个高手 / 长帖 / 爆文 / 方法论 | 外部高手 / 外部长帖镜子型 |

## 8) 与 Momentum Map 的关系

判断原型后，`Momentum-Map.md` 必须补 3 个字段：

- `article_archetype`: 本文采用哪个原型。
- `archetype_rhythm`: 本文哪里快、哪里慢、哪里转。
- `archetype_risk`: 这个原型最容易写坏在哪里。

如果某篇文章同时命中两个原型，必须选一个主原型，另一个只能作为局部模块，不要把两套骨架硬拼。
