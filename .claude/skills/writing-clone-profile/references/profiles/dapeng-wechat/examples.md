## 推荐流正反例（公众号）

### 正例：适合承担推荐流第一入口的标题/包装

| 标题                                                             | 为什么适合推荐流                                                                 | 备注                                |
| ---------------------------------------------------------------- | -------------------------------------------------------------------------------- | ----------------------------------- |
| `YC CEO 开源的 Skill，被大佬们转发爆了，这两个用起来真有奇效！`  | 一眼能看懂读者能带走什么：两个可直接上手的 Skill；工具名、结果感、使用收益都明确 | 适合“技能推荐 / 工具拆解”型流量头条 |
| `我把大神 Karpathy 最新爆火项目，做成了无限进化的内容包装 Skill` | 外部热点被改造成“我的实操 + 你的可带走结果”，不是纯拆解 Karpathy                 | 先给可用结果，再给底层思维          |
| `中文用户必须知道的 Skill 商店`                                  | 资源入口极明确，读者点开前就知道能拿到一个工具集合                               | 典型“资源型底盘”标题                |
| `一个 Skill，解决 90% 的信息图需求`                              | 问题具体，解决方案具体，收益具体；门槛很低                                       | 典型“一个 + 解决大问题”模式         |
| `我用 Claude Code + Skill 帮老婆跑通AI+教育的最小变现闭环`       | 真实案例、具体对象、具体结果、可复用路径都在标题里                               | 案例承载底层思维的正例              |

### 反例：更适合中下游转化，不适合作为推荐流第一入口

| 标题                                                 | 问题                                                                               | 适合位置                             |
| ---------------------------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------ |
| `为什么你总是断更？不是不自律，是没有内容复利工作台` | 概念先行，读者即时收益不够直接；更像作者在重新定义问题，而不是先告诉读者能带走什么 | 适合中下游转化文，不适合承担流量头条 |

### 这一组正反例要提醒什么

- 推荐流不是反对转化，而是先要求“有用”清晰可见
- 同样讲底层思维，先挂 skill / 技巧 / 案例，更容易被点开
- 如果标题更像产品定位名词，而不是读者收益表达，默认要提高警惕
