# Source Quality Notes（dapeng-wechat）

这份文件记录第二轮优化时纳入 profile 资产的素材来源与质量判断。

它不是运行时写稿素材库，而是给后续维护者看的证据索引：哪些素材可以抽规则，哪些只能弱参考，哪些不应该进入风格规则。

## 1) 采纳标准

素材进入 `dapeng-wechat` 的二层资产前，必须至少满足下面 4 条：

1. 是已发布稿、精选代表作、或有 `ai-baseline -> final` 证据链的成稿，不拿随机草稿当稳定风格。
2. 能定位到完整正文或完整对比链路，不只看单句截图、标题或 review 结论。
3. 包含大鹏自己的经历、判断、工具栈、业务数据、项目过程或明确取舍。
4. 抽取的是可泛化机制，不回流一次性事实、截图链接、当日工具版本或临时营销动作。

## 4) 弱样本与不采纳项

| 来源 | 判断 |
| --- | --- |
| `01_项目/内容创作/260410_starter上手指南/` baseline/final | 更偏教程包装与展示样式调整，语义级改写不强，只做弱样本 |
| `03-Reviews/Claim-Ledger.md`、`Lint-Report.md`、`Gate-Check-Report.md` 等 | 是审计材料，不是风格样本；可抽 failure signal，不能直接当正文范例 |
| 只有标题、封面、截图占位、平台关注引导的片段 | 不进入风格规则，避免把平台噪音学进去 |
| 外部作者金句或方法论原文 | 只能借结构、问题意识或节奏，不进入大鹏 voice |
| `03_归档/已发布文章/_用于初始化skill_精选代表作/越是高阶用户...` | 更像短爆点或碎片式表达，不够稳定，不进入主样本池 |
| 视频号口播稿、录制版脚本 | 适合作为口播节奏参考，不作为公众号长文母本 |

## 5) 维护原则

- 新增素材时，先补本文件，再补 `article-archetypes.md`、`opening-question-transition-moves.md` 或 `rewrite-delta-examples.md`。
- 如果一条规则只在单篇出现，先标成候选，不直接写进 `style-profile.md`。
- 如果一条规则连续出现在 2-3 条高质量成稿或人工 diff 里，再考虑升格为稳定 profile 规则。
