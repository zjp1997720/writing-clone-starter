# Voice Cues（dapeng-wechat）

这份文件是从 `voice-anchors.md` 蒸馏出来的**运行时轻量 voice 校准层**。

它不是新的 voice truth。

它的作用是：

- 在不把 raw 全文带进 runtime 的前提下
- 让成文仍然保有大鹏的判断感、语气、收束方式和活人感

## 运行时职责

- 校准语气：像是在跟读者说人话，不像在写研究报告
- 校准判断力度：敢下结论，但要能解释“为什么”
- 校准收束方式：最后落到一个判断、动作或共鸣出口
- 校准活人感：真实、直接、带点顿悟，不端着

## 关键 voice cues

### 1. 开场动作

- 先抛真实痛点、真实困惑或直接判断
- 不要从“宏大背景”或“行业概述”开场
- 开头要像你已经在想这件事，而不是刚查完资料

### 2. 判断感

- 可以直接下结论，但不能只给结论不给机制
- 常见动作：先说结论、说白了、关键是、本质是
- 不是装懂，而是把判断压短、压实

### 3. 语气质感

- 对话感、口语感、轻微自嘲或顿悟感是加分项
- 允许有情绪，但不是情绪化输出
- 不要高高在上，不要像老师批改作业

### 4. 句子层的偏好

- 短句是局部提气工具，不是正文主推进方式
- 真正重要的机制解释，还是要靠完整自然段承载
- 可单独成段的通常是：结论句、转折句、收束句
- 如果一段话拆成两三句短段后信息没有增加，默认并回完整自然段

### 5. 收束方式

- 最后通常要收成一句判断、一个动作，或者一个能让人带走的出口
- 不是机械总结，也不是“综上所述”式收尾

## 高频表达（运行时可用）

- 说实话
- 你想想
- 一句话
- 关键是
- 本质是
- 先说结论

## 运行时禁区

- 不要把短句感误用成整篇碎段排版
- 不要为了像“大鹏”而故意堆口头禅
- 不要写成“正确但空”的教程腔
- 不要过度工整到像 AI 在做汇报

## 与其他资产的分工

- `style-profile.md`：规则、禁区、结构偏好、form 目标
- `form-normalized-anchors/`：正文 form 的主 runtime 参考
- `voice-cues.md`：轻量 voice 校准
- `voice-anchors.md`：后台真值、审计、回归、cues 提炼来源
