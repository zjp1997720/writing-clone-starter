# 上下文充足度评估规范

用于 Gate A：判断当前上下文是否足够直接进入写作。

## 1) 评分槽位（总分 100）

| 槽位                           | 分值 | 判定标准                       |
| ------------------------------ | ---: | ------------------------------ |
| 主题清晰度（topic）            |   15 | 主题边界清楚，避免”大而空”     |
| 读者画像（target_reader）      |   10 | 能描述读者层级与典型场景       |
| 核心立场（stance）             |   15 | 有明确主张，不是中性拼盘       |
| 案例/素材（evidence/examples） |   20 | 至少有 1-2 个可落地案例        |
| 约束条件（constraints）        |   10 | 字数、风格、禁区、是否可写数据 |
| 行动目标（goal/CTA）           |   10 | 读后希望用户做什么             |
| 记忆上下文（memory）           |   10 | 已搜索记忆并找到相关洞察       |
| 实时性（timeliness）           |   10 | 若需要最新数据/趋势，已联网调研 |

## 2) 决策阈值

- `score >= 70`：无需追问，直接写作。
- `50 <= score < 70`：只问 1 个最高杠杆问题。
- `score < 50`：最多问 2 个问题（问完必须继续，不可无限追问）。

## 3) 追问选择规则

只问”补齐后最能提升成文质量”的槽位。优先顺序：

1. 核心立场（stance）
2. 案例素材（evidence/examples）
3. 读者画像（target_reader）
4. 行动目标（goal/CTA）
5. 约束条件（constraints）
6. 主题清晰度（topic）

**注意**：记忆上下文和实时性不需要追问，直接执行搜索即可。

## 4) 提问格式（强制）

每个问题必须包含三段：

1. 缺失项：我现在缺少什么
2. 影响：为什么它会影响成文质量
3. 默认值：若你不回复，我将按什么继续

示例：

"我现在缺少你的核心立场，这会影响文章观点力度。\
如果你暂时不补充，我会按『上下文资产比提示词更重要』作为默认主张继续。"

## 5) 默认值策略（防阻塞）

用户未回复时，直接采用默认值继续：

- 读者默认：对该主题有基本兴趣、希望快速获得可执行建议的读者
- 立场默认：给出明确主张与取舍，避免中性拼盘
- 案例默认：优先用用户当前对话/素材中提供的案例；若缺失，允许使用常识性通用案例（不编造具体数据）
- 目标默认：帮助读者形成一个可立刻执行的小闭环
