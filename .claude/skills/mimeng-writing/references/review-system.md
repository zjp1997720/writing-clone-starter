# 复盘体系：把经验变成能力

## 核心理念

**复盘是把经验变成能力的过程。**

**咪蒙原话："很多时候说我做了两年就有经验了吗？不一定。你没有复盘，你还是没有经验。"**

---

## 三级复盘架构

| 频率     | 内容                    | 产出         | 目的                       |
| -------- | ----------------------- | ------------ | -------------------------- |
| **每日** | 用户留言报告            | 当天反馈分析 | 知道用户喜欢什么、讨厌什么 |
| **每周** | 公号数据分析 + 同行分析 | 周度复盘报告 | 向表现好的同行学习         |
| **每月** | 新媒体趋势分析          | 行业趋势洞察 | 把握行业方向               |

---

## 第1级：每日用户留言报告

### 目的

**每天知道用户对当天文章的真实反馈。**

### 报告内容

**咪蒙原话："今天有60%的人喜欢这个标题，有20%的人觉得开头有戳到，12%的人非常讨厌今天的结尾。"**

| 维度     | 统计内容       | 示例                  |
| -------- | -------------- | --------------------- |
| **标题** | 喜欢比例       | "60%的人喜欢这个标题" |
| **开头** | 被戳到比例     | "20%觉得开头有戳到"   |
| **金句** | 哪句被提到最多 | "第3个金句被截图最多" |
| **结尾** | 讨厌比例       | "12%讨厌今天的结尾"   |
| **整体** | 好评/差评比    | "好评72%，差评28%"    |

### 报告的残酷性

**咪蒙原话："每一天真的是血淋淋的。经常会觉得我自己做的特别得意的梗，然后读者说超无聊，很尴尬。就是每天都看到这个报告都想哭。"**

**关键发现：**

- 你觉得得意的梗，可能读者觉得无聊
- 你没想到的点，可能是读者最喜欢的
- 你的感觉90%可能是错的

### 操作方法

1. **收集留言**：当天文章的所有留言
2. **分类统计**：按维度分类（标题/开头/金句/结尾）
3. **计算比例**：好评/差评的百分比
4. **提炼洞察**：今天做对了什么，做错了什么

## 第2级：每周公号数据分析

### 目的

**向表现好的同行学习，对比自己的数据变化。**

### 报告内容

#### 自己的数据分析

| 指标         | 本周 | 上周 | 变化 | 分析       |
| ------------ | ---- | ---- | ---- | ---------- |
| 平均阅读量   |      |      |      |            |
| 打开率       |      |      |      |            |
| 转发率       |      |      |      |            |
| 留言数       |      |      |      |            |
| 最高阅读文章 |      |      |      | 为什么高？ |
| 最低阅读文章 |      |      |      | 为什么低？ |

#### 同行分析

**咪蒙原话："我很喜欢去分析同行。在我们公司的始末里面都有你们的报告。"**

| 分析内容     | 重点                                   |
| ------------ | -------------------------------------- |
| **同行爆款** | 这周有哪些同行出了爆款？为什么？       |
| **选题对比** | 同样的热点，他们怎么切的？我怎么切的？ |
| **标题对比** | 他们的标题有什么特点？                 |
| **结构对比** | 他们的文章结构有什么可学的？           |

### 2017年复盘案例

**咪蒙做了什么：**

> 2018年了，我们给2017年很多同行点赞前30都做了报告。为什么他们做得好？向同行学习。

## 第3级：每月新媒体趋势分析

### 目的

**把握行业方向，不被时代淘汰。**

### 分析内容

1. **平台变化**：微信、抖音、小红书有什么新动态？
2. **用户变化**：用户的阅读习惯有什么变化？
3. **内容趋势**：什么类型的内容在崛起/衰落？
4. **竞品动态**：头部账号有什么新动作？

### 咪蒙的趋势洞察（2018年演讲）

#### 洞察1：新媒体迟早会变成旧媒体，但观点和故事是永恒刚需

> "从原始社会到现在到未来社会，新观点和新故事的需求是不会被替代的，只会越来越重要。"
>
> "我们不仅需要新故事，我们需要对新故事的看法，对生活的新洞察。"

**启示**：不要追逐形式，要追逐本质。观点和故事是永恒的。

#### 洞察2：观点是引爆点

> "我分析了我所有自己觉得写得好的文章以及数据高的文章，他做对了一点，就是新的观点和新的洞察。"

**启示**：要下更多功夫去提供对生活的新洞察和新看法。

#### 洞察3：不是打开率下降了，而是流量高度集中了

> "用户的要求变高了，打开线和转发线都变高了。"
>
> "大家更愿意去点那些反复被点击的文章，而不是看一个陌生的文章。"

**启示**：分享基因极其重要。一定要给用户一个转发的理由。

#### 洞察4：在AI时代，人要更像人

> "自媒体的本质不是媒体，而是人。"
>
> "一定要让用户感受到你的人格化。"
>
> "在AI时代，我们跟机器人能抗争的是什么？是我们更像人。"

**启示**：不要变成冷冰冰的媒体，要保持人格化和温度。

## 复盘的执行要点

### 1. 必须坚持

**复盘不是一次性的，是每天、每周、每月的常态。**

### 2. 必须量化

**不是"感觉"，而是"数据"。**

- ❌ "感觉这篇写得不错"
- ✅ "阅读量比上周同期高23%，转发率提升15%"

### 3. 必须对比

**对比才能发现差距。**

- 与自己的历史对比
- 与同行的爆款对比

### 4. 必须归因

**知道"是什么"不够，要知道"为什么"。**

- 这篇高了，为什么高？
- 那篇低了，为什么低？
- 可复制的是什么？要避免的是什么？

### 5. 必须迭代

**复盘的目的是迭代，不是总结。**

- 发现问题 → 下次改进
- 发现成功 → 下次复制

## 咪蒙的反思：媒体化的错误

**2017年8月的错误决策：**

> "我其实挺焦虑的，觉得爆款率在下降。然后我就开始分析同行的爆款，觉得有些文章真的写得很好。"
>
> "我做了一个错误的决策，就是把我的公号媒体化了。"
>
> "你会看到2017年8月之后，我的公号上经常都会有'我采访了'、'我调查了'。"
>
> "以前采访完，我会把采访痕迹去掉，像给朋友讲故事那样讲。但后来我把采访痕迹还原了。"
>
> "这是不对的。因为好的公号是什么？就像在每天晚上给朋友讲一个故事，讲一个新看法。当你变成媒体的时候，用户会觉得你的自媒体没有温度了。"

**教训**：

- 自媒体的本质不是媒体，而是人
- 不要因为焦虑就模仿别人的形式
- 保持自己的人格化和温度

---

**终极原则**：复盘是把经验变成能力的过程。没有复盘，做两年也没有经验。有了复盘，做两个月就能超过别人两年。
