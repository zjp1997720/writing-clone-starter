# 审稿质量标准：咪蒙式内容检测清单

## 审稿流程

### 第一轮：黄金三秒测试

**操作**：只读前三屏内容，然后停下。

**检测标准**：

| 检测项       | 合格标准                    | 不合格信号               | 修改方案                     |
| ------------ | --------------------------- | ------------------------ | ---------------------------- |
| **视觉冲击** | 第一句话短、狠、独立成段    | 第一句超过15字或平铺直叙 | 改成短句+感叹号，单独成段    |
| **情绪强度** | 前三句出现至少1个极端情绪词 | 语气平和、中性表达       | 替换为：炸、哭、滚、气死     |
| **悬念设置** | 读完想知道"接下来怎么样"    | 读完觉得"就这样啊"       | 增加反转、提出疑问、制造冲突 |
| **场景具体** | 有具体的人、对话、时间      | 抽象概念、泛泛而谈       | 加入"我的朋友"故事或对话     |

**案例分析**：

❌ **不合格开头**：

```
最近职场压力很大，很多人都在面临各种挑战。
今天我想和大家聊聊职场的那些事。
```

- 问题：太平淡，没有冲击力，像写论文

✅ **合格开头**：

```
我要气炸了。

刚才，我的实习生当着我的面摔了杯子。

她说：凭什么你赚得比我多？
```

- 优点：情绪强烈、场景具体、有冲突

---

### 第二轮：情绪强度检测

**操作**：统计全文的情绪词密度。

**检测标准**：

#### 情绪词密度要求

- **最低标准**：每100字至少1个情绪词
- **优秀标准**：每50字1个情绪词
- **爆款标准**：每30字1个情绪词

**情绪词分类表**：

| 类型       | 词汇库                                 | 使用场景               |
| ---------- | -------------------------------------- | ---------------------- |
| **愤怒类** | 炸、撕、滚、气死、受够了、不要脸、恶心 | 发泄类、对立类文章     |
| **委屈类** | 哭、崩溃、心疼、太难了、凭什么、不公平 | 诉苦类、共鸣类文章     |
| **焦虑类** | 完了、来不及、落后、淘汰、抛弃、输了   | 阶层焦虑、职场焦虑文章 |
| **爽感类** | 爽、杀、打脸、逆袭、报应、活该         | 逆袭类、复仇类文章     |
| **鄙视类** | low、low逼、贱人、圣母、巨婴、蠢       | 标签对立类文章         |

**检测工具**：

统计以下中性词出现次数，**必须全部替换**：

| 中性词   | 替换为     |
| -------- | ---------- |
| 不开心   | 崩溃       |
| 讨厌     | 恶心死了   |
| 喜欢     | 爱到骨子里 |
| 不好     | 烂透了     |
| 很生气   | 气炸了     |
| 有点难过 | 哭到窒息   |
| 不太满意 | 受够了     |

### 第三轮：金句密度统计

**操作**：在手机上预览文章，统计每屏的金句数量。

**检测标准**：

- **最低标准**：每屏至少1个金句
- **优秀标准**：每屏2个金句
- **爆款标准**：每屏3个金句

**金句判定标准**：

一个合格的金句必须满足：

1. **独立性**：脱离上下文也能理解
2. **可传播性**：适合直接发朋友圈
3. **情绪浓度**：包含极端词或绝对化表达
4. **节奏感**：读起来有韵律，朗朗上口

**金句公式库**：

#### 公式1：对仗式

```
[A]，才是最高级的[B]
```

- 例：自律，才是最高级的自由
- 例：发脾气，才是最高级的秀恩爱

#### 公式2：反转式

```
你以为[A]，其实[B]
```

- 例：你以为他在忙，其实他就是不想理你
- 例：你以为是运气不好，其实是你太懒

#### 公式3：宣告式

```
记住：[绝对化判断]
```

- 例：记住：不回微信的男朋友，不是真的爱你
- 例：记住：能用钱解决的，都不是事

#### 公式4：质问式

```
凭什么[X]？
```

- 例：凭什么你要帮他？
- 例：凭什么女生就要懂事？

#### 公式5：否定式

```
别再[错误做法]了，[真相]
```

- 例：别再等他了，他不会来的
- 例：别再努力了，方向比努力更重要

### 第四轮：逻辑自洽检查

**目的**：避免《寒门状元之死》式的硬伤。

**检测项目**：

#### 1. 时间线一致性

**检查**：

- [ ] 文中提到的所有时间是否自洽？
- [ ] 人物年龄与事件时间是否匹配？
- [ ] 引用的产品/事件上线时间是否正确？

**案例警示**：

- ❌ "2013年高考的他，高中时看过2014年才上线的网剧"
- ✅ 如果要用虚构细节，必须模糊时间："那年高考"

#### 2. 地点与场景真实性

**检查**：

- [ ] 提到的学校/公司是否存在？
- [ ] 地点细节是否经得起考证？
- [ ] 如果虚构，是否已经模糊处理？

**安全做法**：

- ✅ "某一线城市的某互联网大厂"
- ✅ "我朋友小美（化名）"
- ❌ "北京字节跳动的产品经理王明"

#### 3. 数据可信性

**检查**：

- [ ] 引用的数据是否有来源？
- [ ] 数据是否夸张到不可信？
- [ ] 如果是编造的，是否会被轻易打脸？

**安全做法**：

- ✅ "有调查显示，超过70%的职场人..."（模糊来源）
- ✅ "我身边10个朋友，8个都..."（个人观察，无法证伪）
- ❌ "2023年北京市平均工资是8万"（可被查证的公开数据，不能乱说）

#### 4. 人物行为合理性

**检查**：

- [ ] 人物的反应是否符合常理？
- [ ] 对话是否太过戏剧化？
- [ ] 是否有明显的逻辑漏洞？

**案例**：

- ❌ "他月薪5000，却开着玛莎拉蒂"（逻辑不通）
- ✅ "他月薪5000，却花3000买了个包"（虽然夸张但可信）

### 第五轮：红线检查

**目的**：避免被封号。

**严禁触碰的红线**：

#### 1. 政治敏感（零容忍）

**禁止**：

- ❌ 涉及国家领导人
- ❌ 涉及政策解读
- ❌ 涉及体制批评
- ❌ 涉及敏感历史事件

**安全边界**：

- ✅ 停留在情感、职场、生活方式
- ✅ 吐槽个人、公司、行业
- ✅ 吐槽社会现象（但不上升到制度）

#### 2. 伪造事实（高风险）

**禁止**：

- ❌ 明确标注"非虚构"但内容是编的
- ❌ 伪造可查证的公开数据
- ❌ 冒用真实公众人物的经历

**安全做法**：

- ✅ 虚构故事明确标注"本文纯属虚构"
- ✅ 或者在账号简介说明"故事源于生活但高于生活"
- ✅ 使用化名、模糊时间地点

#### 3. 极端煽动（中风险）

**禁止**：

- ❌ 煽动仇富到阶级对立
- ❌ 煽动性别对立到极端
- ❌ 教唆违法行为（如报复、伤害）

**安全边界**：

- ✅ "不要和渣男纠缠，分手就完了"（情感建议）
- ❌ "遇到渣男就该XX他"（教唆违法）

#### 4. 侮辱人格（中风险）

**禁止**：

- ❌ 针对特定真实个人的人身攻击
- ❌ 地域歧视
- ❌ 职业歧视到侮辱程度

**安全边界**：

- ✅ 吐槽抽象的"渣男"、"奇葩老板"（群体标签）
- ❌ 点名道姓骂某个真实的人

---

### 第六轮：传播力评估

**操作**：问自己3个问题。

#### 问题1：读者为什么要转发？

**合格答案**（至少满足一个）：

- [ ] 表达态度（"我也这么想"）
- [ ] 展示立场（"我是这样的人"）
- [ ] 寻求认同（"求大家评评理"）
- [ ] 发泄情绪（"太解气了"）
- [ ] 警示他人（"你们也要小心"）

**不合格信号**：

- 读完只觉得"哦，有道理"但没有行动冲动
- 没有强烈的情感共鸣

**改进方案**：

- 增加"转发给XXX的人"引导语
- 在结尾加一句煽动性的行动号召

#### 问题2：读者会在哪里卡住？

**检测方法**：

- 找一个不了解背景的人读一遍
- 记录他们皱眉、停顿、倒回去重读的地方

**常见卡点**：

- 段落太长（视觉压迫）
- 逻辑跳跃（前后不连贯）
- 术语太多（看不懂）
- 情绪突然降温（节奏断裂）

**修改原则**：

- 砍掉所有让人停顿的地方
- 用更短的句子
- 用更具体的场景

#### 问题3：这篇文章的核心情绪是什么？

**要求**：用一个词概括。

**合格答案**：

- 愤怒
- 委屈
- 焦虑
- 爽感
- 恐惧

**不合格答案**：

- "复杂"（说明情绪不聚焦）
- "平静"（说明没有情绪）
- "理性"（这不是咪蒙风格）

**修改方案**：

- 确定一个核心情绪
- 删掉所有干扰这个情绪的内容
- 强化这个情绪的表达

## 综合评分表

完成所有检查后，给文章打分：

| 检测项   | 权重 | 评分标准           | 得分   |
| -------- | ---- | ------------------ | ------ |
| 黄金三秒 | 25%  | 0-10分，不合格<6分 | \_\_\_ |
| 情绪强度 | 20%  | 0-10分，不合格<7分 | \_\_\_ |
| 金句密度 | 20%  | 0-10分，不合格<6分 | \_\_\_ |
| 逻辑自洽 | 15%  | 0-10分，有硬伤=0分 | \_\_\_ |
| 红线检查 | 10%  | 碰红线=整篇作废    | \_\_\_ |
| 传播力   | 10%  | 0-10分，不合格<6分 | \_\_\_ |

**总分计算**：加权平均

**发布标准**：

- 总分≥80分：可以发布
- 总分60-80分：需要修改后发布
- 总分<60分：推倒重写

## 快速审稿口诀

```
三秒抓人没？（开头）
情绪够狠没？（强度）
金句够密没？（密度）
逻辑能圆没？（自洽）
红线碰了没？（风险）
转发动力有没？（传播）
```

**六个"没"都过了，就可以发。**

## 审稿后的修改优先级

如果时间有限，按以下优先级修改：

### P0（必改）

1. **红线问题**：碰了立刻删
2. **逻辑硬伤**：能被打脸的细节立刻改

### P1（强烈建议改）

3. **开头三秒**：决定80%的阅读完成率
4. **情绪强度**：决定转发率

### P2（有时间就改）

5. **金句密度**：提升传播力
6. **传播力优化**：增加转发引导

---

**终极原则**：审稿不是挑错，是确保每个读者都能被击中。一篇合格的咪蒙式文章，应该让90%的目标读者看完后想转发。
