# business-whiteboard

商业白板手绘信息图风 - 高信息密度、第一眼可见手绘感

## Design Aesthetic

商业手绘白板信息图风。线条可以干脆、流畅、圆润，但手绘痕迹必须在第一眼就可见，而不是放大后才看出来。正文默认优先高信息密度信息图，而不是低密度氛围插图。逻辑箭头指向明确，绝不交叉重叠。整体应像人手在白板或纸面上画出来再整理过，而不是软件直接排版生成的标准商业图表或 clean slide 模板。

## Background

- Color: 白纸或浅米白纸质感 (#FFFFFF / #FAF8F0)
- Texture: 可见但克制的纸面或白板表面感；不能完全无材质到像数字纯白画布

## Color Palette

| Role       | Color           | Hex     | Usage                                                            |
| ---------- | --------------- | ------- | ---------------------------------------------------------------- |
| Background | Pure White      | #FFFFFF | 纯白背景                                                         |
| Primary    | Deep Gray/Black | #2A2A2A | 主线条，统一使用深灰色或黑色                                     |
| Accent     | Morandi Blue    | #8FA3B0 | 唯一强调色（低饱和度单色），仅用于涂色高亮最核心的节点或关键箭头 |

## Visual Elements

- 干脆、流畅、圆润的线条
- 明显可见的轻微抖动、轻微不完美、marker 笔触感
- 结构化图解（金字塔、流程图、四象限等）
- 高信息密度的信息图布局（对比图、决策图、模块图、流程图）
- 有强手写感的中文关键词标注
- 层级分明的字号递减
- 明确的逻辑箭头（指向清晰，不交叉）
- 核心主题用大号手写体居中或在顶部呈现
- 次要节点用精炼关键词标注
- 适度的 doodle 式图标、角色或符号，只要不破坏商业感
- 可见的纸面 / 白板表面感
- 框体、箭头、连接线、分隔线都要抗拒 machine-perfect appearance

## Style Rules

### Do

- 正文图优先高信息密度，但层级必须清楚，可扫读、可截图
- 留白服务层级，不服务空洞感；不要用大面积空白替代信息承载
- 线条干脆、流畅、圆润
- 线条、框体、箭头、图标都保留强可见的手绘感和轻微不完美
- 严格控制文字数量，以图解意而非以字填图
- 中文文字保留明显手写感，像白板笔或马克笔写出来的字，并且第一眼不能像电脑排版字
- 逻辑箭头指向明确
- 仅用一种低饱和度单色强调（莫兰迪蓝、灰绿或赭石色）
- 排版符合职场精英阅读习惯（从左到右、从上到下）
- 视觉焦点明确（核心主题突出）
- 看起来像商业手绘信息图，而不是模板化企业图表
- 手绘感必须 first-glance visible，而不是 second-glance subtle
- 如果必须在“过于干净”和“略微不规则”之间取舍，优先保留可读前提下的人手活力

### Don't

- 多色混杂（红黄绿并存）
- 装饰性涂鸦（星星、圆点、波浪线等无意义装饰）
- 元素交叉或重叠
- 为了填满画布而堆砌元素
- 空洞氛围图、纯装饰图、低信息密度大留白图（正文默认）
- 规整得像企业模板图、PPT 图表、无手绘痕迹的标准 icon system
- 中文文字像标准印刷体或海报字体
- 复杂的先锋艺术表达或冲击力设计
- 在空白处添加无意义的装饰性涂鸦
- 太干净，导致第一眼像 polished slide-template infographic
- 布局看起来像 software-typeset first，hand-drawn second
- 图标、箭头、框体太标准，像 vector icon library 拼出来的
- 几何精确到像软件自动对齐后的商业模板图

## Density Boundary

- 这份 style 定义的是正文默认模式：高信息密度信息图
- 这份 style 定义的是正文默认模式：商业手绘的高信息密度信息图
- 这份 style 默认要求：第一观感先是“手绘知识图解”，再是“规整商业信息图”
- WeChat cover 不受这条默认密度合同约束
- 如果用户显式传入其他 style，优先尊重用户指定风格

## Best For

商业分析、逻辑框架、知识结构、战略规划、流程图解、概念模型、职场培训、管理咨询、商业演示
